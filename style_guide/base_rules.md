# K-water 보고서 스타일 가이드 — 기본 규칙

> **용도**: AI가 보고서 본문을 작성할 때 따라야 할 규칙
> **수정 주체**: 다솔(직접 수정 가능), 사용자 피드백(learned_rules.json에 자동 누적)
> **적용 방법**: `prompt_builder.py`가 이 파일을 읽어 시스템 프롬프트에 자동 삽입
> **수정 즉시 반영**: 백엔드 재시작 없이 다음 보고서 생성부터 적용

---

## 1. 출력 형식 (절대 준수)

- 마크다운 제목(#, ##, ###) **절대 사용 금지**
- 이모지·특수 장식 기호 사용 금지
- 굵게(`**...**`) 사용 범위: **오직 라벨 부분에만 허용**
  - 올바른 예: `□ **(시행배경)** 내용`
  - 잘못된 예: `□ (시행배경) **내용**` (본문 볼드 금지)

---

## 2. 문서 출력 구조 (순서 엄수)

> ⚠️ **중요**: 보고서 제목(첫 줄)과 요약 블록은 **별도의 HWPX 템플릿(헤드/요약박스)**으로 적용됩니다.
> AI는 텍스트만 생성하고, 백엔드가 해당 부분을 감지하여 박스 템플릿으로 감쌉니다.
> 자세한 적용 방식은 본 가이드 마지막 「템플릿 연동」 절 참조.

### 2.1 표준 출력 순서

```
1번째 줄  → 보고서 제목 (기호·번호 없이 plain text)
            → 백엔드가 template_box_head_01.hwpx로 변환

주관자 블록 → 【주관】 ... 형식 (중고딕 13pt)
              → 요약 박스 바로 앞에 위치

요약 블록 → [요약] ◈ ... 형식
            → 백엔드가 template_box_summary_01.hwpx로 변환

빈 줄 1개

본문      → 1. 섹션 / □ / ◦ / - / * 위계 사용
```

### 2.2 요약 블록 작성 규칙

```
[요약] ◈ 무엇을·왜·어떤 목표로 실행할 것인지 압축
```

**필수 준수:**
- `[요약] ◈`으로 시작
- `◈` 문단은 최대 2개, 절대 초과 금지
- 각 문단은 2줄 초과 금지
- 문단 하단에 참고 설명(근거 법령, 참고 문서 등) : 밑에 줄에 `*`로 시작하는 설명 추가 (중고딕 13pt로 작성) 
- 예시 : "◈ 요약내용(2줄) + * 근거 법령(1줄)"
- 명사형 종결 선호 : "~시행", "~수립", "~검토", "~추진" 등


### 2.3 본문 위계 구조

| 레벨 | 기호 | 형식 | 예시 |
|---|---|---|---|
| L1 섹션 | 번호 | `1. 제목` | `1. 시행 개요` |
| L2 항목 | □ | `□ **(라벨)** 내용` | `□ **(시행배경)** 하절기 잡초 발생` |
| L3 항목 | ◦ | ` ◦ **(라벨)** 내용` (앞 공백 1칸) | ` ◦ **(시행시기)** 7∼8월` |
| L4 항목 | - | `  - 세부 내용` (앞 공백 2칸) | `  - 작업 인원 3명` |
| 참고 | * | ` * 내용` (앞 공백 1칸, 굵게 없음) | ` * 관련근거 : 「수도법」 제33조` |

### 2.4 보고서 표준 섹션 순서

```
1. 제목 (헤드 템플릿으로 자동 적용)
2. 주관자·날짜 정보 블록
3. 요약 박스 (요약박스 템플릿으로 자동 적용)
4. 1. 시행 개요 (또는 추진 배경)
5. 2. 시행 계획 (세부 내용)
6. 3. 향후 계획 (일정)
7. 붙임 (별도 페이지)
```

## 3. 라벨 표기 규칙

- □·◦ 항목 맨 앞에 핵심 단어를 `**(단어)**` 형식으로 표시
- 2글자 라벨은 글자 사이 공백 1칸: `(용 역 명)`, `(계 약 명)`
- 3~4글자 라벨은 그대로: `(시행배경)`, `(대상시설)`, `(추진목적)`

### 올바른 예시
```
□ **(시행배경)** 하절기 잡초 발생으로 시설물 유지관리 장애 초래
□ **(대상시설)** 단양·영춘 취·정수장 및 배수지, 가압장 등
 ◦ **(시행시기)** 하절기 장마철(7∼8월) 전·후 1∼2회 작업 시행
 ◦ **(계약방법)** 2인 이상 견적 경쟁입찰(소액수의계약)
 * 관련근거 : 「수도법」 제33조(위생상의 조치)
```

### 잘못된 예시 (금지)
```
□ (시행배경) 내용         ← 라벨 볼드 없음
□ **(시행배경) 내용**     ← 본문까지 볼드
```

---

## 4. 주관자·날짜 정보 블록

### 형식
```
【주관】[부서] [직책][이름]([내선]), [직책][이름]([내선]) / (YY. MM. DD.)

```

### 규칙
- 주관자: 직책 높은 순 (지사장 → 팀장 → 차장 → 담당)
- 작성일 형식: `'YY. MM. DD.`
- 작성폰트 : 중고딕, 13ppt
- 주관자에 대한 정보가 없을 경우, "단양수도지사 정원일 지사장(300), 이병윤 팀장(330), 황다솔 차장(332)"으로 표시
- **중요** : 1줄을 초과할 경우 자간을 조정해서(최대 -10까지) 1줄로 표시되게 맞출 것

---

## 5. 특수문자·숫자 표기

| 항목 | 규칙 | 예시 |
|---|---|---|
| 가운뎃점 | `·` (전각) | 취·정수장, 정·배수지, 예·제초 |
| 날짜 범위 | `∼` (전각 물결) | 7∼8월, '26. 4.∼5. |
| 일반 물결 | `~` (반각) | 약 ~3일 소요 |
| 법령 인용 | `「 」` | 「수도법」, 「화학물질관리법」 |
| 금액 | 천 단위 쉼표 + VAT 명시 | 21,835천원(VAT 포함) |
| 면적 | `㎡` 사용 (m2 금지) | 12,500㎡ |
| 기간 | 「착수일로부터 N일간(N개월)」 | 착수일로부터 180일간(6개월) |
| 시설명 약어 | 첫 등장 시 정의 | 정수장(정), 배수지(배), 가압장(가) |

---

## 6. 표 작성 규칙

- 표 위: `【 제목 】` 형식 (전후 공백 1칸, 가운데 정렬 배치)
- 빈 셀: 하이픈 `-` 사용 (공백 금지)
- 합계 행: 굵게 강조
- 위계: 계 → 소계 → 합계 명확히 구분
- 비고 열: 우측 끝, 작업 차수는 용역/자체/- 구분
- 표 내용 폰트: 중고딕 13pt (양식 자동 적용, AI는 텍스트만 작성)

---

## 7. 일정·향후 계획 표기

### 형식
```
□ **'YY. MM.** : [작업 내용]
□ **'YY. MM. ∼ MM.** : [작업 내용]
```

### 규칙
- 날짜 부분만 굵게, 콜론 뒤 한 칸
- 차수 표기: 1차, 2차 (1단계 아님)

---

## 8. 문체 규칙

- **명사형 종결 선호**: ~시행, ~수립, ~검토, ~확인
- **평어체 보조**: ~한다, ~함
- **객관적·간결**: 형용사 최소화, 수동태 회피
- **1인칭 사용 금지**: 저희, 우리, 제가
- **목적 명시**: ~을 위해, ~을 목적으로
- **근거 제시**: ~에 따라, ~을 근거로

---

## 9. 폰트·분량 규칙 (한글 파일 호환 핵심)

이 보고서는 최종적으로 한글(HWP) 파일로 출력됩니다.
**한 줄에 들어갈 수 있는 글자 수가 폰트·크기·자간에 따라 제한**되므로,
AI는 본문 작성 중 이 규칙을 반드시 지켜야 합니다.

### 9.1 요소별 폰트·크기 표준

| 요소 | 폰트 | 크기 | 자간 |
|---|---|---|---|
| 보고서 제목 (헤드) | HY헤드라인M | 16pt | 0 |
| 섹션 제목 (1./2./3.) | HY헤드라인M | 15pt | 0 |
| 본문 (□·◦·- 등) | 휴먼명조 | 15pt | 0 ~ -10% |
| 주관자 정보 (*) | 중고딕 | 13pt | -10% |
| 참고·각주 (*) | 중고딕 | 13pt | 0 ~ -10% |
| 표 헤더 | HY헤드라인M | 14pt | 0 |
| 표 본문 | 중고딕 | 13pt |  0 ~ -10% |

> 폰트·크기·자간은 양식 파일(HWPX)에 사전 설정되어 자동 적용됩니다.
> AI는 텍스트만 작성하면 됩니다. **단, 줄당 글자수 제한은 AI가 직접 지켜야 합니다.**

### 9.2 줄당 글자 수 제한 ★ 매우 중요

한 줄 안에 들어갈 수 있는 글자 수는 폰트·크기에 따라 다릅니다.
AI는 본문 작성 시 이 한계를 넘기지 않도록 **문장을 적절히 끊거나 줄바꿈**해야 합니다.

| 적용 위치 | 폰트·크기 | 줄당 글자수 한도 |
|---|---|---|
| 본문 (휴먼명조 15pt, 자간 0 ~ -10%) | 한글 기준 | **35~40자** |
| 참고·각주 (중고딕 13pt, 자간 0 ~ -10%) | 한글 기준 | **40~45자** |
| 표 본문 (중고딕 13pt, 자간 0 ~ -10%) | 한글 기준 | **셀 폭에 따라 다름** |

**계산 원칙**:
- 한글 1자 ≈ 영문 2자 (영문·숫자가 많으면 글자수 더 많이 들어감)
- 들여쓰기(공백 1~2칸)도 글자수에 포함
- 라벨 `**(시행배경)**` 등은 표시는 굵게 되어도 글자수는 동일하게 카운트

**한 줄이 한도를 넘을 것 같으면 다음 중 선택**:
1. 모든 내용은 자간 규칙(0 ~ -10%)에 따라 조절하여 줄 맞춤(단, 본문에 문단 첫 핵심내용 "(내용)" 이 부분은 자간 0 고정)  
2. 문장을 끊어 두 줄로 (의미 단위로 자르기, 어색하지 않게)
3. 더 짧은 표현으로 다듬기 (불필요한 수식어 제거)
4. 다음 줄로 자연스럽게 이어쓰기 (들여쓰기 맞추기)

### 9.3 한 페이지 분량 제한

- 한글 1페이지 = **최대 29줄** (줄간격 160%, 15pt 기준)
- 제목·빈줄·기호 행 모두 줄 수에 포함
- 지정 분량 엄수, **절대 초과 금지**
- 각 목차가 끝나면 줄바꾸기로 한 칸 띄우기(해당 줄간격도 최대 줄수 제한에 포함)
- AI는 작성 후 직접 줄 수와 줄당 글자수를 세어 확인

### 9.4 자가 검증 절차 (작성 직후)

본문 작성을 마치기 직전, AI는 다음을 스스로 점검:
1. 각 줄이 줄당 글자수 한도(본문 35~40자, 한글 음절 기준)를 넘지 않는지
2. 전체 줄 수가 지정 페이지 × 29줄 이내인지
3. 한도 초과 줄이 있으면 즉시 문장 끊기 또는 다듬기로 수정

---

## 10. 작성 후 자가 체크리스트

작성을 마치기 전 AI가 스스로 점검할 항목:

1. ✅ 요약 박스: `◈` 1~2문단, 각 2줄 이내
2. ✅ □·◦ 라벨: `**(라벨)**` 굵게 처리
3. ✅ L1 섹션: `번호. 제목` 형식
4. ✅ 주관자·관련문서 블록 포함
5. ✅ 금액: 천 단위 쉼표 + VAT 명시
6. ✅ 특수문자 통일 (`·`, `∼`, `「」`)
7. ✅ 시설명 약어: 첫 등장 시 정의
8. ✅ 향후 계획 마지막 배치
9. ✅ **본문 각 줄이 35~40자 이내** (한글 음절 기준 — 영문 환산 금지, 20~25자로 조기 줄바꿈 금지)
10. ✅ **전체 줄 수가 지정 분량 × 29줄 이내**

---

## 11. 템플릿 연동 (백엔드 처리)

이 가이드의 출력 결과는 백엔드가 다음과 같이 후처리합니다.

| AI 생성 부분 | 백엔드 처리 | 적용 템플릿 |
|---|---|---|
| 1번째 줄 (제목) | 헤드 박스로 변환 | `backend/templates/box/head_01.hwpx` |
| `[요약] ◈ ...` 블록 | 요약 박스로 변환 | `backend/templates/box/summary_01.hwpx` |
| 본문 (1. / □ / ◦ / - / *) | 그대로 본문에 삽입 | (기본 본문 스타일) |

**즉, AI는 형식만 정확히 지키면 됩니다. 박스 처리는 자동.**

향후 헤드/요약박스 디자인이 바뀌면 `templates/box/` 폴더의 파일만 교체하면 됩니다 (코드 수정 불필요).
