# K-water 보고서 스타일 가이드 — 한글 서식 규칙

> **용도**: 보고서의 폰트·크기·자간·여백 등 시각적 서식 정의
> **적용 방식**: HWPX 양식 파일에 사전 설정 → 본문 텍스트만 치환
> **수정 주체**: 다솔(서식 표준 변경 시 양식 파일 자체를 교체)

---

## 1. 폰트·크기 표준

| 요소 | 폰트 | 크기 | 비고 |
|---|---|---|---|
| 보고서 제목 (헤드) | HY헤드라인M | 16pt | 굵게, 헤드 박스 템플릿이 적용 |
| 섹션 제목 (1./2./3.) | HY헤드라인M | 15pt | 굵게 |
| 본문 | 휴먼명조 | 15pt | 자간 -10% 이내 |
| 하위 항목 (□, ◦, -) | 휴먼명조 | 15pt | 본문과 동일 |
| 참고·각주 (*) | 중고딕 | 13pt | * 기호로 시작 |
| 표 헤더 | 휴먼명조 | 13pt | 굵게, 가운데 정렬 |
| 표 본문 | 휴먼명조 | 13pt | |

---

## 2. 줄간격 및 페이지 설정

- 줄간격: **160%** (15pt 본문 기준)
- 1페이지 최대 줄 수: **29줄**
- 페이지 여백 (예시값, 실제값은 양식 파일 따름):
  - 위·아래: 20mm
  - 좌·우: 25mm

---

## 3. 계층 구조 기호 규칙

| 단계 | 시작 기호 | 들여쓰기 | 비고 |
|---|---|---|---|
| L1 (대분류) | `1.`, `2.`, `3.` | 없음 | 숫자 + 점 + 공백 |
| L2 (소분류) | `□` | 없음 | 라벨 굵게 처리 |
| L3 (세부) | `◦` | 공백 1칸 | 라벨 굵게 처리 |
| L4 (상세) | `-` | 공백 2칸 | 라벨 없음 |
| 참고·각주 | `*` | 공백 1칸 | 굵게 없음 |

---

## 4. 박스 템플릿 적용

다음 두 요소는 **별도 HWPX 박스 템플릿**으로 처리됩니다.

### 4.1 헤드 박스 (보고서 제목)
- 파일: `backend/templates/box/head_01.hwpx`
- 적용 대상: 보고서 1번째 줄
- 시각 효과: 굵은 헤더, 박스 테두리, 배경색 등 (양식 파일에 사전 설정)

### 4.2 요약 박스 ([요약] ㅁ ...)
- 파일: `backend/templates/box/summary_01.hwpx`
- 적용 대상: `[요약] ㅁ`으로 시작하는 블록
- 시각 효과: 회색 배경, 좌측 강조선, 들여쓰기 등 (양식 파일에 사전 설정)

> 향후 박스 디자인 변경이 필요하면 위 두 파일을 한컴오피스에서 직접 수정하면 됩니다.
> 코드 수정 없이 디자인만 바뀝니다.

---

## 5. AI가 신경 쓰지 않아도 되는 것

다음 항목들은 양식 파일과 백엔드가 자동 처리하므로, AI는 **본문 텍스트 작성에만 집중**합니다.

- 폰트 선택 (HY헤드라인, 휴먼명조, 중고딕)
- 폰트 크기·자간
- 줄간격·페이지 여백
- 헤드/요약 박스의 시각적 디자인
- 표 셀 너비·정렬

AI가 신경 쓸 것은 **텍스트 자체의 구조와 표기 규칙**(`base_rules.md`)뿐입니다.
